    PHP Super Globals

    9. PHP Superglobals

    PHP superglobals are built-in global arrays that provide access to various types of data, such as form submissions, session data, cookies, and server information. These variables are always accessible, regardless of scope, and can be used throughout your scripts.


    9.1 $_GET

    The $_GET superglobal is used to collect form data after submitting an HTML form using the GET method. It is also used to retrieve query string variables from the URL.


    • Syntax Example:
      • // URL: example.com?name=John&age=25
      echo $_GET['name']; // Outputs: John
      echo $_GET['age']; // Outputs: 25
    • Explanation:

      • The $_GET superglobal accesses the data passed in the URL query string.

    9.2 $_POST

    The $_POST superglobal is used to collect form data after submitting a form with the POST method. It is typically used when sending data through the HTTP request body, providing more security than $_GET.


    • Syntax Example:
      • // Assuming a form submits 'name' and 'email'
      echo $_POST['name']; // Outputs: John
      echo $_POST['email']; // Outputs: johndoe@example.com
    • Explanation:

      • The $_POST superglobal accesses the data sent through the POST request method.

    9.3 $_REQUEST

    The $_REQUEST superglobal is a combined array that contains data from $_GET, $_POST, and $_COOKIE. It allows you to access form data regardless of the method used (GET or POST).


    • Syntax Example:
      • // Accessing data from both GET and POST
      echo $_REQUEST['name']; // Can come from GET, POST, or COOKIE
    • Explanation:

      • The $_REQUEST superglobal provides access to variables from multiple sources, making it versatile for handling form submissions.

    9.4 $_SESSION

    The $_SESSION superglobal is used to store session variables, which persist across multiple pages during a user's session. Session data is stored on the server.


    • Syntax Example:
      • session_start();
      $_SESSION['user'] = 'John Doe';
      echo $_SESSION['user']; // Outputs: John Doe
    • Explanation:

      • The $_SESSION superglobal is used to store user-specific data that can be accessed on subsequent pages.

    9.5 $_COOKIE

    The $_COOKIE superglobal is used to access cookies sent by the client's browser. Cookies are small pieces of data stored on the user's device and are sent with every request to the server.


    • Syntax Example:
      • echo $_COOKIE['user']; // Outputs the value of the 'user' cookie
    • Explanation:

      • The $_COOKIE superglobal allows you to retrieve data that is stored as cookies on the user's device.

    9.6 $_FILES

    The $_FILES superglobal is used for handling file uploads. It contains information about the uploaded files, such as their names, sizes, and types.


    • Syntax Example:
      • if ($_FILES['file']['error'] == 0) {
      echo $_FILES['file']['name']; // Outputs the name of the uploaded file
      }
    • Explanation:

      • The $_FILES superglobal stores data related to uploaded files, making it easy to handle file uploads in PHP.

    9.7 $_SERVER

    The $_SERVER superglobal provides information about the server environment, including request headers, server information, and paths.


    • Syntax Example:
      • echo $_SERVER['SERVER_NAME']; // Outputs the server name
    • Explanation:

      • The $_SERVER superglobal contains server-related information, such as the server name and script location.

    9.8 $_ENV

    The $_ENV superglobal is used to retrieve environment variables that provide information about the system's environment.


    • Syntax Example:
      • echo $_ENV['PATH']; // Outputs the system path
    • Explanation:

      • The $_ENV superglobal provides access to environment variables that store system-level data.

    9.9 Conclusion

    PHP superglobals are an essential part of working with data and managing user interactions. By understanding how to use superglobals such as $_GET, $_POST, $_SESSION, and others, you can handle a wide range of scenarios in PHP programming.
