    PHP Introduction to Object-oriented Programming

    5. PHP Introduction to Object-Oriented Programming (OOP)

    Object-Oriented Programming (OOP) is a programming paradigm in PHP that focuses on using objects and classes. It allows you to create more structured, reusable, and maintainable code by organizing your code into "objects" that can represent real-world entities.


    5.1 Key Concepts in OOP

    There are several key concepts in OOP that form the foundation of object-oriented programming:

    • Classes: A class is a blueprint for creating objects. It defines the properties and methods that an object will have.
    • Objects: An object is an instance of a class. Objects hold data and can perform actions using methods defined in their class.
    • Encapsulation: The practice of bundling data (properties) and methods (functions) together in a class. Encapsulation allows for access control to properties, preventing direct modification from outside the class.
    • Inheritance: Inheritance allows a class to inherit properties and methods from another class, promoting code reuse and extension.
    • Polymorphism: Polymorphism allows objects of different classes to be treated as objects of a common parent class, enabling flexibility in code.
    • Abstraction: Abstraction hides complex implementation details and provides a simple interface to interact with objects.
    5.2 Why Use OOP in PHP?
    • Code Reusability: Once a class is written, it can be reused to create multiple objects without rewriting code.
    • Maintainability: OOP helps in organizing code into classes and objects, making it easier to maintain and update the codebase.
    • Modularity: Code is organized into classes, which can be individually modified or updated without affecting the entire application.
    • Real-World Modeling: OOP allows developers to represent real-world entities like cars, animals, users, etc., using objects.
    5.3 Basic Syntax of Classes and Objects

    The basic syntax for creating a class and instantiating an object is as follows:

    • Syntax Example:
      • class Car {
           // Properties
           public $color;
           public $model;

           // Methods
           public function drive() {
               echo "Driving...";
          }

      }

      $myCar = new Car();
      $myCar->color = "red";
      $myCar->model = "Toyota";
      $myCar->drive(); // Outputs: "Driving..."
    • Explanation:

      • In this example, the Car class is defined with two properties: color and model, and one method: drive(). We then create a new Car object and assign values to its properties.

    5.4 Access Modifiers in OOP

    PHP provides three access modifiers for controlling the visibility of class properties and methods:

    • public: The property or method is accessible from outside the class.
    • private: The property or method is only accessible within the class where it is defined.
    • protected: The property or method is accessible within the class and by subclasses.
    5.5 Inheritance in PHP

    Inheritance allows one class to inherit the properties and methods of another. This promotes code reuse and allows for the extension of existing functionality.

    • Syntax Example:
      • class Animal {
           public function speak() {
               echo "Animal sound";
          }
      }

      class Dog extends Animal {
           public function speak() {
               echo "Bark";
          }
      }

      $dog = new Dog();
      $dog->speak(); // Outputs: "Bark"
    • Explanation:

      • In this example, the Dog class inherits from the Animal class. The Dog class overrides the speak() method to output "Bark" instead of "Animal sound".

    5.6 Conclusion

    Object-Oriented Programming in PHP allows developers to write cleaner, more modular, and reusable code. By using concepts like classes, objects, inheritance, and polymorphism, you can create powerful, scalable applications. OOP helps model real-world scenarios and makes your code easier to maintain and extend over time.
