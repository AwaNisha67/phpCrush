    PHP Abstract Classes

    11. PHP Abstract Classes

    Abstract Classes in PHP are classes that cannot be instantiated on their own and must be inherited by other classes. An abstract class can contain abstract methods, which are methods without implementation. Abstract methods must be implemented by any class that extends the abstract class. Abstract classes allow you to define a common interface for all derived classes, while leaving the implementation details to the subclasses.


    11.1 Defining an Abstract Class

    An abstract class is defined using the abstract keyword. The abstract class can have both regular methods (with implementations) and abstract methods (without implementations). Any class that extends an abstract class must implement all of its abstract methods, unless that class is also abstract.

    • Syntax Example:
      • abstract class Animal {
           abstract public function makeSound();

           public function eat() {
               echo "Eating...";
          }
      }

      class Dog extends Animal {
           public function makeSound() {
               echo "Bark!";
          }
      }

      // Instantiating the Dog class
      $dog = new Dog();
      $dog->makeSound(); // Output: Bark!
      $dog->eat(); // Output: Eating...

      In this example, the Animal class is abstract and defines an abstract method makeSound(). The Dog class inherits from Animal and implements the makeSound() method. The eat() method is a regular method defined in the abstract class and does not need to be overridden by the child class.

    11.2 Key Points About Abstract Classes
    • Abstract Methods: These are methods declared in the abstract class without an implementation. They must be implemented by any non-abstract class that extends the abstract class.
    • Cannot Instantiate Abstract Classes: Abstract classes cannot be instantiated directly. You must create an instance of a subclass that implements all the abstract methods.
    • Abstract and Non-Abstract Methods: An abstract class can have both abstract methods (methods without implementation) and non-abstract methods (methods with implementation).
    • Child Classes: Any class that extends an abstract class must implement all of its abstract methods, unless the child class is also abstract.
    • Abstract Class Usage: Abstract classes are often used to define a blueprint for other classes, where the common interface is shared, but the implementation is specific to each child class.
    11.3 Example: Abstract Class with Abstract and Concrete Methods

    In this example, the abstract class has both abstract and concrete methods. The child class must implement the abstract method but can use the inherited concrete method as is.

    • Syntax Example:
      • abstract class Vehicle {
           abstract public function start();

           public function stop() {
               echo "Vehicle has stopped.";
          }
      }

      class Car extends Vehicle {
           public function start() {
               echo "Car is starting!";
          }
      }

      $car = new Car();
      $car->start(); // Output: Car is starting!
      $car->stop(); // Output: Vehicle has stopped.

      In this example, the Vehicle class is abstract and contains both an abstract method start() and a concrete method stop(). The Car class implements the start() method and inherits the stop() method without modification.

    11.4 When to Use Abstract Classes
    • Defining Common Interface: Use abstract classes when you want to define a common set of methods that must be implemented by child classes.
    • Partial Implementation: Abstract classes are useful when you want to provide some default behavior (i.e., non-abstract methods) while enforcing that child classes implement certain methods themselves.
    • Code Reusability: Abstract classes help you avoid duplicating code by allowing shared functionality to be implemented in the abstract class and used by all subclasses.
