    PHP Destructor

    9. PHP Destructor

    A destructor in PHP is a special function that is automatically called when an object is destroyed or when it is no longer in use. Destructors are used to perform any cleanup tasks, such as closing database connections, freeing up resources, or releasing memory before the object is removed from memory.


    9.1 Defining a Destructor

    In PHP, destructors are defined using the __destruct method. This method is called when an object is destroyed, either explicitly or when the script finishes execution.

    • Syntax Example:
      • class MyClass {
           // Destructor method
           public function __destruct() {
               echo "Object destroyed";
          }
      }

      This example shows how to define a destructor method __destruct() inside a class.

    9.2 When the Destructor is Called

    The destructor is automatically called when an object is destroyed or when the script finishes execution. You do not need to manually call the destructor method.

    • Syntax Example:
      • class Person {
           public $name;
           public $age;

           // Constructor method
           public function __construct($name, $age) {
               $this->name = $name;
               $this->age = $age;
          }

           // Destructor method
           public function __destruct() {
               echo "Goodbye, " . $this->name . ".
      ";
          }
      }

      // Creating an object
      $person1 = new Person("John", 30);

      // Destructor is automatically called when the object is destroyed
      unset($person1); // Output: Goodbye, John.

      In this example, when the unset() function is called on the object $person1, the destructor is automatically invoked, and the message "Goodbye, John." is printed.

    9.3 Key Points about Destructors
    • Automatic Invocation: The destructor is automatically called when an object is destroyed, either explicitly using unset() or when the script execution ends.
    • No Arguments: A destructor method cannot accept parameters. It is called automatically without passing any arguments.
    • Resource Cleanup: Destructors are typically used for resource cleanup, such as closing file handles, database connections, or releasing memory.
    • Cannot be Overloaded: Like constructors, PHP does not support destructor overloading. There can only be one destructor per class.
    • Destruction at Script End: If an object is not explicitly destroyed using unset(), it will be destroyed at the end of the script's execution, when the object is no longer in scope.
